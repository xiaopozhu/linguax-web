This guide helps you map side buttons on macOS without overcomplicating the setup.

## What You Can Map in LinguaX

- `Button 4`
- `Button 5`
- `MR` / `ML` (when supported by your device profile)
- click / double click / long press / directional drag gestures

## Recommended Setup Order

1. Pick one high-value action first (for example: back, mission control, app shortcut).
2. Map it to one side button.
3. Use it for one full work session.
4. Add a second mapping only if the first one feels natural.

## Keep Mappings Stable

- avoid assigning too many actions to one button
- avoid overlapping mappings across multiple tools
- use app-scoped overrides only where global behavior is not enough

## Good First Mappings

- Browser Back / Forward
- Workspace switch action
- Open launcher / quick command
- Repeated app shortcut (for your editor or design tool)

## Troubleshooting Quick Checks

- confirm accessibility permission is granted
- verify no duplicate mappings in other utilities
- test one button with one gesture first
